IVI Shared Components

The IVI Foundation develops and maintains the IVI shared components, a series of free software components that facilitate IVI driver development and ensure a consistent experience for end users. They must be installed on any computer that uses IVI drivers. IVI driver installers are required to verify the presence of the shared components before installing.

The following sections provide a brief overview of some of the most important shared components.

IVI Shared Component Installer

The IVI Foundation provides an installer for the IVI shared components. The IVI installer:

  • Is the only IVI-compliant way to install the shared components (third parties are not allowed to develop custom installation programs for these components)
  • Is included in the Keysight I/O Libraries Suite installation package
  • Is also available for download from the Shared Components page on the IVI Foundation web site
  • Creates a directory structure to house the shared components as well as IVI drivers themselves

The installation folder is referred to as the IVI install directory (<IviInstallDir>) and is typically C:\Program Files\IVI Foundation\IVI.

IVI Configuration Store

The IVI Configuration Store (called the Config Store) serves as a central repository for IVI driver configuration information. The Config Store is an XML file installed with the IVI shared components in <IviInstallDir>\Data\ IviConfigurationStore.xml.

IVI driver installers are required to populate the Config Store with information such as the instrument classes supported, the instrument models for which the driver is designed, and the physical channel names understood by the driver. End users can:

  • Add information to the Config Store that describes how they intend to use IVI drivers in their system
  • Specify logical names for IVI driver sessions and virtual names for channels

For more information, see:

  • IVI Configuration Store for a more detailed explanation
  • IVI-3.5: Configuration Server Specification available on the IVI Foundation's Specifications page

IVI Configuration Server  

The IVI Foundation discourages users and driver developers from directly manipulating the Config Store because its file (IviConfigurationStore.xml) is highly self-referential and easily corrupted if modified directly. To enable users to modify the Config Store, the Foundation provides a COM component known as the IVI Configuration Server.

The Config Server is installed automatically by the IVI shared component installer. For more information on how to use the IVI Configuration Server, see IVI-3.5: Configuration Server Specification on the IVI Foundation web site.
