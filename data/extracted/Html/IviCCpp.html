Visual C++

This section describes how to use the KtNA IVI-C driver with Visual C++.  For information on:

  • Initialization not specific to C++, see Initializing the Driver
  • Programming with IVI drivers, see the Getting Started Guide on the IVI Foundation website

Accessing the Driver

To access the driver, all IVI-C driver programs (and your Visual Studio project) must:

  • Reference the proper import library:  KtNA.lib
  • Include the proper header file: KtNA.h
  • Prefix function calls with "KtNA_"

To use the KtNA driver, perform the following steps in Visual Studio.

  1. In Solution Explorer, right-click on the project and select Properties.
  2. In the property pages dialog, expand the Linker node and select Input.
  3. In the Additional Dependencies field, enter:
     "KtNA.lib"
  4. Click OK.
  5. In the main application source file (.cpp), add the following line to the top of the file:
     #include "KtNA.h"
  6. Specify the path to your driver's library (.lib) and header (.h) files:
    1. Select Tools > Options > Projects and Solutions > VC++ Directories.
    2. Enter the following paths. For:
      • Include files, add:
          C:\Program Files\IVI Foundation\IVI\Include (Win32 and x64)
      • Library files, add:
          C:\Program Files\IVI Foundation\IVI\Lib_\msc (Win32)
          C:\Program Files\IVI Foundation\IVI\Lib_x64\msc (x64)

      3. Note that these paths:

      • Start with C:\Program Files (x86) for 32-bit operating systems
      • Also can be entered in the Project Properties dialog in the Configuration Properties, C++ and Linker panes

Initializing the Driver

To initialize the driver:

  1. Call KtNA_InitWithOptions to either:
    • Establish an I/O connection to an instrument (often referred to as an "I/O session") 
    • Put the driver in simulation mode
  2. Ensure you also call KtNA_close at the end of your program (as required by IVI specifications) to:
    • Avoid unpredictable driver behavior
    • Ensure that any resources held by the driver are properly released

The driver is typically initialized using a resource descriptor, often a VISA resource descriptor, as shown below.

C++ Copy Code
ViSession session;
ViStatus status;
status = KtNA_InitWithOptions("MyVisaAlias", VI_TRUE, VI_TRUE, "", &session); // "MyVisaAlias" is a VISA alias for your instrument.
status = KtNA_close(session);

Initializing with Options

This example shows how to pass IVI-defined initialization options and driver-specific options to the Initialize function.

C++ Copy Code
// If true, this will query the instrument model and fail initialization 
// if the model is not supported by the driver
ViBoolean idQuery = VI_FALSE;

// If true, the instrument is reset at initialization
ViBoolean reset = VI_FALSE;

// Setup IVI-defined initialization options
ViConstString standardInitOptions = 
  "Cache=true, InterchangeCheck=false, QueryInstrStatus=true, RangeCheck=true, RecordCoercions=false, Simulate=false";

status = KtNA_InitWithOptions("MyVisaAlias", idQuery, reset, standardInitOptions, &session); // "MyVisaAlias" is a VISA alias for your instrument.
status = KtNA_close(vi);

Accessing Attributes

IVI-C driver attributes are accessed through a set of IVI-defined accessor functions. Each attribute type has two accessor functions: Get for reading attribute values, and Set for writing attribute values.

The standard Get attribute accessors for reading attribute values are:

  • GetAttributeViInt32
  • GetAttributeViInt64
  • GetAttributeViReal64
  • GetAttributeViBoolean
  • GetAttributeViString

Correspondingly, the standard Set attribute accessors for writing attribute values are:

  • SetAttributeViInt32
  • SetAttributeViInt64
  • SetAttributeViReal64
  • SetAttributeViBoolean
  • SetAttributeViString

Each attribute accessor takes an attribute ID that uniquely identifies the attribute to access. These attribute IDs are:

  •  #define'd constants listed in the KtNA.h header file
  • Documented in the Attributes by Name section of the IVI-C Driver Reference

The following example shows how to use attribute accessors to read and write IVI-C driver attribute values.

Note: This code is for illustration only and may not apply to your driver.

C++ Copy Code
ViStatus status;

// Set the value of an attribute
status = KtNA_SetAttributeViReal64(vi, VI_NULL, KTNA_ATTR_VOLTAGE_RANGE, 20.0);

// Get the value of a property 
ViReal64 voltageRange;
status = KtNA_GetAttributeViReal64(vi, VI_NULL, KTNA_ATTR_VOLTAGE_RANGE, &voltageRange);

// Set the value of a property (enum)
// Note use of the #define'd constant for the attribute value
status = KtNA_SetAttributeViInt32(vi, VI_NULL, KTNA_ATTR_TEMP_TRANSDUCER_TYPE, KTNA_VAL_THERMOCOUPLE);

Complete Example

To see complete, working example programs, go to Windows Start Menu > All Programs > Keysight Instrument Drivers > IVI-C KtNA 3.0.8.

C++ Copy Code
#include "KtNA.h"

void main()
{
  ViSession session;
  ViStatus status;
  ViChar str[128];
  ViInt32 ErrorCode;
  ViChar ErrorMessage[256];

  // Initialize the driver
  status = KtNA_InitWithOptions("MyVisaAlias", VI_TRUE, VI_TRUE, "", &session); // "MyVisaAlias" is a VISA alias for your instrument.
  if(status)
  {
     // Initialization failed
     KtNA_GetError(session, &ErrorCode, 255, ErrorMessage);
     printf("** InitWithOptions() Error: %d, %s\n", ErrorCode, ErrorMessage);
     printf("\nDone - Press Enter to Exit");
     getchar();  
     return;
  }
  printf("Driver Initialized \n");
 
  // Read and output a few attributes
  status = KtNA_GetAttributeViString(session, "", KTNA_ATTR_SPECIFIC_DRIVER_PREFIX, 127, str);
  assert(status == VI_SUCCESS);
  printf("DRIVER_PREFIX:      %s\n", str);
  status = KtNAl_GetAttributeViString(session, "", KTNA_ATTR_SPECIFIC_DRIVER_REVISION, 127, str);
  assert(status == VI_SUCCESS);
  printf("DRIVER_REVISION:    %s\n", str);
  status = KtNA_GetAttributeViString(session, "", KTNA_ATTR_SPECIFIC_DRIVER_VENDOR, 127, str);
  assert(status == VI_SUCCESS);
  printf("DRIVER_VENDOR:      %s\n", str);
  status = KtNA_GetAttributeViString(session, "", KTNA_ATTR_SPECIFIC_DRIVER_DESCRIPTION, 127, str);
  assert(status == VI_SUCCESS);
  printf("DRIVER_DESCRIPTION: %s\n", str);
  status = KtNA_GetAttributeViString(session, "", KTNA_ATTR_INSTRUMENT_MODEL, 127, str);
  assert(status == VI_SUCCESS);
  printf("INSTRUMENT_MODEL:   %s\n", str);
  status = KtNA_GetAttributeViString(session, "", KTNA_ATTR_INSTRUMENT_FIRMWARE_REVISION, 127, str);
  assert(status == VI_SUCCESS);
  printf("FIRMWARE_REVISION:  %s\n", str);
  
  
  // TODO: Exercise driver functions and attributes
  
  
  // Check instrument for errors
  ErrorCode = 1;
  printf("\n");
  while(ErrorCode)
  {
     status = KtNA_error_query( session, &ErrorCode, ErrorMessage);
     assert(status == VI_SUCCESS);
     printf("error_query: %d, %s\n", ErrorCode, ErrorMessage);
  }

  // Close the session
  status = KtNA_close(session);
  if(status)
  {
     KtNA_GetError(session, &ErrorCode, 255, ErrorMessage);
     printf("** Close() Error: %d, %s\n", ErrorCode, ErrorMessage);
  }
  else
     printf("Driver Closed \n");
}
