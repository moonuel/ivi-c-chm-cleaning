Programming the Instrument

After you have initialized the driver, you are ready to use all the functionality that the driver provides. Note that:

  • All driver functions begin with KtNA_, the instrument prefix.
  • All driver attributes begin with KTNA_ATTR_.

Example: Accessing Functionality

The following example shows how to access the KtNA driver functionality.

C++Copy Code
#include "KtNA.h"
void main()
{
  ViStatus status;
  ViSession session;
  ViReal64 reading;
  
  status = KtNA_InitWithOptions("MyVisaAlias", VI_TRUE, VI_TRUE, "", &session);
  status = KtNA_ReadEx(session, 2000, 3, &reading);
  status = KtNA_close(session);
}

Setting and Retrieving Instrument Settings (Attributes)

IVI-C instrument settings are called attributes. Each driver uses constants to define a set of attributes.  For example, if the instrument has a voltage setting, the IVI-C attribute constant might be KTNA_ATTR_VOLTAGE.

Each attribute has an associated type. IVI drivers use attributes set and get functions, one set of functions per type, to set or retrieve attribute values.  For example, if voltage is a real, then the following functions would be used to set or retrieve the value of the voltage setting:

  • Set: KtNA_SetAttributeViReal64()
  • Retrieve: KtNA_GetAttributeViReal64()

This driver provides functions to get and set attributes associated with the following types:

  • ViInt32 – Signed 32-bit integer
  • ViInt64 – Signed 64-bit integer
  • ViReal64 – 64-bit floating-point number
  • ViBoolean – Boolean value
  • ViSession (pointers) – Instrument driver session handle
  • ViString – Sequence of characters

Note that all attributes are scalars, except for strings.
