KtNA Repeated Capabilities

This topic describes the supported repeated capabilities and repeated capability names for the KtNA driver.  For a detailed explanation of repeated capabilities, including repeated capability styles, physical names, and dynamic repeated capabilities, see Repeated Capabilities.

Repeated Capabilities

The KtNA driver supports the following repeated capabilities:

  • Channel
  • Window
  • Trace
  • Measurement
  • AuxConnector
  • SourcePort
  • Step
  • CalibrationSession
  • ECalModule
    •

The details for each repeated capability are described in the following table.

Repeated Capability IVI-C Style Dynamic Physical Names
Channel Parameter Yes
Window Parameter Yes
Trace Parameter Yes
Measurement Parameter Yes
AuxConnector Parameter Yes
SourcePort Parameter Yes
Step Parameter Yes
CalibrationSession Parameter Yes CalibrationSession1
ECalModule Parameter Yes

Dynamic Repeated Capability instances and physical names are determined at run time depending on instrument model's actual capabilities. The following table lists the naming conventions used with this driver's dynamic repeated capabilites.

Repeated Capability Physical Names Naming Convention
Channel Channel1 to Channel500.
AuxConnector AuxConnector1 and AuxConnector2.
CalibrationSession CalibrationSession1 to CalibrationSession250.
ECalModule ECalModule1 and ECalModule254.
Measurement Measurement1 to Measurement2000.
SourcePort Port1 to Port50.
Step Step1 to Step1000.
Window Window1 to Window500.
Trace Trace1 to Trace100.
