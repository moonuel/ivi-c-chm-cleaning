Instrument-Specific Capabilities

 The IVI Foundation defines two general types of drivers:

  • Instrument-specific drivers, which are created for a specific instrument (or instrument family) and expose the full functionality of the instrument, including functionality that may also be available in class-compliant capabilities.
  • Class drivers, which are implementations of the IVI-defined C API for a particular instrument class and are used in conjunction with class-compliant instrument-specific drivers. Class drivers enable instrument interchangeability by exposing only the class-compliant methods and properties available in the instrument.

Modern instruments typically possess much more functionality than is represented in the IVI instrument class specifications. To allow end users to access the full capability of the instrument, the driver must expose interfaces that are specific to that instrument alone. These are known as instrument-specific capabilities.

Instrument-specific capabilities are obviously not interchangeable. End users who program against a driver's instrument-specific interfaces typically do so to access functionality that is not available in the class-compliant interfaces. For complex instruments (such as spectrum analyzers), the driver's instrument-specific capabilities often provide far more functionality than the class-compliant capabilities provide.

Although IVI does not dictate the syntax and semantics of instrument-specific interfaces, guidelines must be followed in the design of the interfaces if the driver is to be IVI compliant. Most of these rules are laid out in IVI-3.4: API Style Guide available in the IVI Specifications.
