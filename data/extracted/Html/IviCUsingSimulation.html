Simulating an Instrument

An IVI driver can operate without connecting to an actual instrument by using simulation mode. When simulation is enabled, the driver does not perform instrument I/O. For output parameters that represent instrument data, the driver functions return simulated values.

To enable simulation, either:

  • Pass the "simulate=VI_TRUE" option string parameter to the InitWithOptions function
  • Set the SIMULATE attribute to VI_TRUE or true

The IVI-C driver contains a SIMULATE attribute. Note that you can turn simulation on, if you initialize a driver with it off. However, if you initialize an IVI-C driver with simulation turned on, you will get an error if you attempt to later turn it off. This is because you cannot initiate a live I/O session with an instrument after you have called the InitWithOptions function.

Using The Option String

The fourth and last parameter to the InitWithOptions function is the option string (see InitWithOptions in the Reference section for more details). To turn simulation on, include the following in the option string:

simulate=VI_TRUE

For example, the InitWithOptions call might look like this:

KtNA_InitWithOptions("MyVisaAlias", VI_TRUE, VI_TRUE, "Simulate=VI_TRUE", &session);

When the simulate option is set VI_TRUE, the value of the resource descriptor parameter ("MyVisaAlias") is ignored because no I/O connection is made with the instrument (as is required by IVI-3.2: Inherent Capabilities Specification).

Specifying the Instrument Model to Simulate

Many drivers support more than one instrument model. Applications can select the model to simulate by specifying the model in the option string. If not specified, the driver uses a default instrument model for simulation.

Specify the model to simulate at runtime during the initialization call. The complete InitWithOptions call becomes (for example):

KtNA_InitWithOptions("MyVisaAlias"::INSTR", VI_TRUE, VI_TRUE, "Simulate=VI_TRUE, DriverSetup= Model=E5080A", &session);

Note: "MyVisaAlias" is a VISA alias for your instrument. If you set the model to a value not supported by the driver, the driver returns an error.

Limitations of Simulation

Simulation is designed to allow the driver to be used without an instrument or any I/O software or hardware. However, simulation is not intended to fully emulate the instrument. Be aware of the following differences:

  • Many instruments have status systems, and these are not live. For instance, you can set Enable and Transition registers. However, queries of Condition and Event registers always return a fixed value (typically zero).
  • Parameter range checking or coercion is typically not done, unless done in the driver. Most drivers depend on the instrument to perform range checking and coercion of parameter values. In simulation mode, this does not occur. For example, setting a frequency parameter to 1E99 will be accepted without error in simulation, while an instrument would return a "parameter out of range" error.
  • KtNA_error_query typically returns no error.
  • Measurement data is not live. Note that:
    • Most measurement queries return a fixed or random value.
    • Most parameter queries return the value that was set.
    • Modular instruments typically validate all settings in simulation mode.

The Driver Session Simulate Attribute

A DriverSession in the IVI Configuration Store contains properties for the items that can be set with the option string parameter. The name of a driver session (or of a logical name that points to a driver session) can be used as the first parameter to InitWithOptions. If the SIMULATE attribute in the driver session is true, the driver is initialized in simulation mode.

Note: Keysight does not provide a Config Store editor. See IVI Configuration Store for more information.
