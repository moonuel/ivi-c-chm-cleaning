MATLAB

MATLAB provides rich support for building applications with IVI-C drivers. This section describes how to use the KtNA IVI-C driver with MATLAB.  For information on:

  • Initialization not specific to MATLAB, see Initializing the Driver
  • Programming with IVI drivers, see the Getting Started Guide on the IVI Foundation website

Requirements

The following components must be installed:

  • MATLAB R2015a (or newer) 32- or 64-bit version
  • MATLAB Instrument Control Toolbox

Creating A MATLAB Instrument Driver

The KtNA IVI-C driver must be converted to a MATLAB Instrument Driver (MID) using the makemid function. Follow these steps:
  1. Open MATLAB.
  2. From the Command Prompt, execute:
    MATLAB Copy Code
    makemid('KtNA', strcat(toolboxdir('instrument'),'\instrument\drivers\KtNA_IVI-C.mdd'), 'IVI-C');
This creates a MID named KtNA_IVI-C.mdd. This MID:
  • Uses the installed KtNA IVI-C driver
  • Is saved in the MATLAB Instrument Control Toolbox\Instrument Drivers folder and accessible to your MATLAB KtNA driver application programs

Note: you must re-run makemid to update the MATLAB Instrument Driver when you either:

  • Update to a new version of the KtNA IVI-C driver, or
  • Update your driver applications to a new release of MATLAB

See the MATLAB mkmid help for more information.

Writing MATLAB Driver Application Programs

Before accessing the various driver functions and attributes to perform desired tasks, your program must first:

  1. Create an instrument device object using the MATLAB icdevice function. The instrument device object retuned by icdevice includes the:
    • VISA resource name for connecting to the instrument
    • IVI-C driver initialization option settings
  2. Connect the device object to the instrument using the MATLAB connect(obj) function.

At the end, your program must: 

  1. Disconnect from the instrument using the MATLAB disconnect(obj) function.
  2. Delete and clear the instrument device object using the MATLAB delete(obj) and clear functions.

Example

MATLAB Copy Code
% Create instrument device object.  "MyVisaAlias" is a VISA alias for your instrument.
resourceName = 'MyVisaAlias';
initOptions = 'QueryInstrStatus=true, Simulate=true, DriverSetup= ';
KtNA = icdevice('KtNA_IVI-C.mdd', resourceName, 'optionstring', initOptions);

% Connect to the instrument using the device object created above
connect(KtNA);
disp('Connected to instrument device: KtNA');

% Perform driver tasks...

% Close and delete the driver device object 
disconnect(KtNA);
delete(KtNA);
clear KtNA;
For additional information, see the MATLAB help for the icdevice, connect, disconnect, delete, and clear functions.

Using MATLAB Instrument Driver Functions and Properties

MATLAB Instrument Driver Syntax

The syntax for accessing the MATLAB instrument driver functions and properties contained in your program's instrument device object is similar to the IVI-C driver hierarchy documented in the Heirarchy section of the KtNA IVI-C Programming Reference.

The instrument device object returned by the icdevice function represents the root node of the IVI-C hierarchy. Use the following syntax to access any function or property in the instrument device object.

  • Property: <Object>.<NodeName>.<Property_Name>
  • Function: invoke(<Object>.<NodeName>, <FunctionName>, ...)

In this syntax:

  • <Object> is the device object variable returned by icdevice
  • <NodeName> is the folder name concatenated with any sub folder names containing the desired function or property
  • <Property_Name> is the name of the desired property (case insensitive)
  • <FunctionName> is the name of the desired function (case insensitive)

Note: The MATLAB function and property names used by the instrument device object do not use the IVI-C driver prefix "KtNA_" or "KtNA_ATTR_" shown in the help file and are case insensitive. Do not include any spaces in the node path or function names. Any spaces or non-alpha-numeric characters in property (attribute) names must be replaced with the underscore (_) character.

See examples in the following sections.

Data Types

The MATLAB instrument device object properties and function parameters of numeric or string types may use matching variable or literal data values directly. Boolean may be set as 0 or false, 1 or true. They return 0 or 1. For items of defined enumeration types in the IVI-C driver, the MATLAB device object uses the integer values listed in this document for each function or attribute using enumerated types. See examples in the following sections.

Note: Some examples may be from other drivers and are not supported by the KtNA driver.

Getting and Setting Properties

Property values may be read or assigned using the = operator with the instrument device object hierarchical path and property name. You do not need to use the "Attribute Accessors" group functions to access properties in the MATLAB instrument device object. For example, using an instrument device object named "KtNA":

MATLAB Copy Code
model = KtNA.InherentIviAttributesInstrumentIdentification.Instrument_Model;
KtNA.System.I_O_Timeout = 5000;
timeoutVal = KtNA.System.I_O_Timeout;

KtScope.Trigger.TriggerType = 1;  % 1 = KTSCOPE_VAL_EDGE_TRIGGER

Properties in Repeated Capabilities

Before setting or getting a property that is defined on a repeated capability in the IVI-C driver, you must set the active instance using the RepCapIdentifier property at the root of the MATLAB instrument device object. For example, using an instrument device object named "KtScope", the following code would enable the trace on channel 2:

MATLAB Copy Code
KtScope.RepCapIdentifier = 'Channel2';
KtScope.Channel.Channel_Enabled = 1;

Note: Not all drivers support repeated capabilities. For more information, see:

  • Assessing Repeated Capabilities (if present) for information on your driver's repeated capabilities, including valid names
  • The documentation for attributes, which indicates whether the attribute is in a repeated capability

Executing Functions

Functions are executed using the MATLAB invoke function with the instrument device object hierarchical path and function name. If a function is defined on a repeated capability, it includes a parameter for the repeated capability instance name. For example, using an instrument device object named "KtNA":

MATLAB Copy Code
invoke(KtNA.Utility, 'Reset');
[code, message] = invoke(KtNA.Utility, 'errorquery');
invoke(KtNA.InstrumentSpecificStatus, 'StatusSetRegister', 12, 128);       % 12 = STATUS_REGISTER_STANDARD_EVENT_ENABLE
[value] = invoke(KtNA.InstrumentSpecificStatus, 'StatusGetRegister', 11);  % 11 = STATUS_REGISTER_STANDARD_EVENT

inputImpedance = 50;
maxFreq = 200e6;
invoke(KtScope.ConfigurationChannel, 'ConfigureChannelCharacteristics', 'Channel2', inputImpedance, maxFreq);

Complete Example

The following code demonstrates a simple console application that creates and initializes the driver, reads some properties, checks the instrument for errors, and closes the driver.

For complete, working example programs, go to Windows Start Menu > All Programs > Keysight Instrument Drivers > IVI-C KtNA 3.0.8.

MATLABCopy Code
% Creates a MATLAB instrument device object, reads a few Identification
% attributes, and checks the instrument error queue.
% May include additional instrument specific functionality.
% 
% Runs in simulation mode without an instrument.
disp(blanks(1)');
disp('  ML_Example1');

% Create and install the MATLAB Instrument Driver.  This may be executed here or from the MATLAB
% Command Window and only needs to be done once after installing the Keysight IVI-C driver.
% makemid('KtNA', strcat(toolboxdir('instrument'),'\instrument\drivers\KtNA_IVI-C.mdd'), 'IVI-C');

% Create the instrument device object.  Edit resourceName and options as needed.  Resource is ignored if option Simulate=true
% "MyVisaAlias" is a VISA alias for your instrument.
resourceName = 'MyVisaAlias';
initOptions = 'QueryInstrStatus=true, Simulate=true, DriverSetup= ';
KtNA = icdevice('KtNA_IVI-C.mdd', resourceName, 'optionstring', initOptions);

% Connect to the instrument using the device object created above
connect(KtNA);
disp('Connected to instrument device: KtNA');

% Print a few IIviDriver.Identity properties
disp(['Prefix:          ', KtNA.InherentIviAttributesDriverIdentification.Specific_Driver_Prefix]);
disp(['Revision:        ', KtNA.InherentIviAttributesDriverIdentification.Specific_Driver_Revision]);
disp(['Vendor:          ', KtNA.InherentIviAttributesDriverIdentification.Specific_Driver_Vendor]);
disp(['Description:     ', KtNA.InherentIviAttributesDriverIdentification.Specific_Driver_Description]);
disp(['InstrumentModel: ', KtNA.InherentIviAttributesInstrumentIdentification.Instrument_Model]);
disp(['FirmwareRev:     ', KtNA.InherentIviAttributesInstrumentIdentification.Instrument_Firmware_Revision]);
simulate = KtNA.InherentIviAttributesUserOptions.Simulate;
if simulate == true
    disp('Simulate:        True');
else
    disp('Simulate:        False');
end
disp(blanks(1));

% TODO: Exercise driver functions and properties
invoke(KtNA.Utility, 'Reset');

% Check instrument for errors
errorNum = -1;
errorMsg = ('');
disp(blanks(1)');
while (errorNum ~= 0)
    [errorNum, errorMsg] = invoke(KtNA.Utility, 'ErrorQuery');
    disp(['ErrorQuery: ', num2str(errorNum), ', ', errorMsg]);
end

% Close and delete the driver device object 
disconnect(KtNA);
delete(KtNA);
clear KtNA;
disp('Done');
