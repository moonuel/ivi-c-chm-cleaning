Modifying the Driver

While you can build and redeploy the driver with relative ease, the process is easier if you understand the details explained here.

Modifiying the Implementation

The IVI.NET driver's C# source code in the Source directory implements the driver methods and properties by calling into a compiled C++ dll via the bridge code in the SwigWrappr directory. To modify the implementation you may replace the generated bridge calls with your custom code and use the following special bridge methods to directly send SCPI commands and queries to the instrument:

  • pCppApi->System()->WriteBytes(std::vector<uint8_t>) - Writes an array of bytes to the instrument
  • pCppApi->System()->WriteString(String) - Writes a string to the instrument
  • std::vector<uint8_t> pCppApi->System()->ReadBytes() - Reads an array of bytes response from the instrument
  • String pCppApi->System()->ReadString() - Reads a string response from the instrument
    Note: You may also use the above methods in your driver application programs via the driver's viWrite and viRead functions. In some cases this may be a better alternative than modifying the driver source code.

For example to change the SCPI sent by a FREQUENCY_CENTER attribute, edit the attribute in Source\KtDriver.cpp, replacing the code with your code as shown:
C++ Copy Code
ViStatus _VI_FUNC KtDriver_get_FREQUENCY_CENTER(ViSession Vi, ViConstString RepCapIdentifier, ViReal64* AttributeValue)
{
	ValidateSession(Vi);
	ObtainLock(Vi);
	CheckForNull(Vi, AttributeValue);
	auto status = VI_SUCCESS;
	try
	{
		GetCppApi();
        //*AttributeValue = pCppApi->Frequency()->GetCenter(); // Original driver code

		pCppApi->System()->WriteString("FREQuency:CW?");
		auto retVal = pCppApi->System()->ReadString();
		*AttributeValue = stof(retVal);
	}
	catch (const exception& e)
	{
		status = TranslateException(Vi, e);
		return status;
	}
    status = PollInstrumentErrors(Vi);
  
    return status;
}

ViStatus _VI_FUNC KtDriver_set_FREQUENCY_CENTER(ViSession Vi, ViConstString RepCapIdentifier, ViReal64 AttributeValue)
{
	ValidateSession(Vi);
	ObtainLock(Vi);
	auto status = VI_SUCCESS;
	try
	{
		GetCppApi();
        //pCppApi->Frequency()->SetCenter(AttributeValue); // Original driver code

		pCppApi->System()->WriteString("FREQuency:CW " + to_string(AttributeValue));
	}
	catch (const exception& e)
	{
		status = TranslateException(Vi, e);
		return status;
	}
    status = PollInstrumentErrors(Vi);
  
    return status;
}

Changing the Version

If you want to change the driver version, edit all instances of the version number in the files DriverInfo.cpp and Versoin.rc2.
The version format is Major.Minor.Build.Revision; integers must be between 0 and 32767.

Changing Driver Attributes

The driver source is installed with the same driver attributes that were used to build the installed driver dll. Keysight recommends the following changes to the Versoin.rc2 file if you are rebuilding the driver and deploying it.

Attribute Recommendation
Comments Do not change.
FileDescription Append "(Modified by <your company name>)" to the description.
CompanyName Change to your company name.
LegalCopyright Append a copyright for your company.
LegalTrademarks Modify as needed for your company.
FileVersion
ProductVersion 		Change as discussed in the versioning section above.

Installation

When you determine how to deploy the rebuilt driver dll, consider the following:

  • The easiest way to deploy the driver is to install it in the same location as the programs that call it. However, this is only practical if the driver will be used by one application, or for very informal debugging.
  • If the driver will be used more extensively, it should be installed as described in IVI-3.17: Installation Requirements Specification , which describes how drivers should be installed. In this case you only need to install the rebuilt dll required by the driver.
