KTNA_ATTR_X_SYSTEM_CALIBRATE_ALL_IFBW Attribute
CalibrateAll  See Also 

Sets and returns the IFBW in Hz for a Cal All calibration.

Attribute Tree Node: \KtNA\Instrument Specific\ClassicCommands\System\CalibrateAll\IFBW
Declaration: KtNA.h
Implementation: KtNA.dll

Parameters

Vi

The ViSession handle that you obtain from the IviDriver_init or IviDriver_InitWithOptions function. The handle identifies a particular instrument session.

RepCapIdentifier

Must be VI_NULL or an empty string. This attribute is not defined on a repeated capability.

AttributeID

Must be KTNA_ATTR_X_SYSTEM_CALIBRATE_ALL_IFBW.

AttributeValue (GetAttribute)

Returns the current value of the attribute. The user must specify the address of a variable that has the same data type as the attribute.

AttributeValue (SetAttribute)

The value to which to set the attribute.

Remarks

•The list of valid IF Bandwidths is different depending on the VNA model. If an invalid number is specified, the VNA will round up to the closest valid setting.
•Default 1 kHz.
•The following IFBW values are common to all models:
1 | 2 | 3 | 5 | 7 | 10 | 15 | 20 | 30 | 50 | 70 | 100 | 150 | 200 | 300 | 500 | 700 | 1k | 1.5k | 2k | 3k | 5k | 7k | 10k | 15k | 20k | 30k | 50k | 70k | 100k | 150k| 200k | 280k | 360k | 600k | 1M | 1.5M | 2M | 3M | 5M | 7M | 10M | 15M.
•For the highlighted IFBW settings (1 MHz and above):

•7 MHz to 15 MHz settings are available ONLY with DSP version 5.0 and above.
•The primary use for IFBW > 600 KHz is for wideband pulsed and pulse profile measurements. They do NOT provide faster sweep speeds for non-pulsed measurements.
•A slight shift (1dB or more) in Log Mag traces may be seen when switching in and out of these bandwidths.
•Available in Step sweep mode only - NOT available in Analog sweep.

•The IFBW is limited to 600 kHz when performing Swept IMD measurements even if the Wide IF path is selected.
