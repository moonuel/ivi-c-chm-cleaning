VBA in Excel

This section describes how to use the KtNA IVI-C driver with Visual Basic for Applications (VBA) in Excel. For information on:
  • Initialization not specific to C++, see Initializing the Driver
  • VBA macro programming in Excel, see the Excel VBA Reference and VBA Language Reference from Microsoft
  • Programming with IVI drivers, see the Getting Started Guide on the IVI Foundation website

Importing the Wrapper and Creating the Module

The driver installer includes one or more VBA wrapper files for the KtNA IVI-C driver. The wrapper provides access to the driver functions, attributes, and defined constants from VBA. To access the driver wrapper functionality, you must import the wrapper files into the Excel VBA Editor for your application. To view the VBA Editor in Excel, click the Visual Basic button on the Developer tab (add it to ribbon if not shown), or press Alt+F11.

VBA code is entered and stored in modules in the VBA Editor. Modules must have a least one subroutine for your VBA code. A VBA project can have one or more modules.

To use the KtNA VBA wrapper and create a module, follow these steps in the VBA Editor window.

  1. Select File > Import File.
  2. Import the KtNA wrapper from <Program Files>\IVI Foundation\IVI\Include\KtNA.bas.
    Note: Large drivers may have additional .bas wrapper files. Import all of them.
  3. Select Insert > Module to create a new module.
  4. Enter the following code  to contain your program in the displayed module edit window:

     Sub MygProram()

     End Sub

To view the VBA wrapper defined function declarations and constants, go to Project Explorer, open the Modules folder, and double-click KtNA.

Note: Wrapper functions and constants use the same names as IVI-C driver functions and attributes. However, the wrapper function parameters are standard VBA types instead of the VISA types used with the IVI-C driver.

Initializing the Driver

Your program must:

  • Call KtNA_InitWithOptions to either:
    • Establish an I/O connection to an instrument (often referred to as an "I/O session") 
    • Put the driver in simulation mode
  • Call KtNA_close at the end of your program (as required by the IVI specifications) to:
    • Avoid unpredictable driver behavior
    • Ensure that any resources held by the driver are properly released

The driver is typically initialized using a resource descriptor (often a VISA resource descriptor or alias) as shown below.

VBA Copy Code
     Dim session As Long
    Dim status  As Long

    status = KtNA_InitWithOptions("MyVisaAlias", True, True, "", session)  ' "MyVisaAlias" is a VISA alias for your instrument.
    <!-- Driver task code -->
    status = KtNA_close(session)

Initializing with Options

This example shows how to pass IVI-defined initialization options and driver-specific options to the Initialize function.

VBA Copy Code
    Dim session As Long
    Dim status  As Long

    ' VISA resource descriptor or alias.  Ignored if option Simulate=true.
    Const resourceDescriptor As String = "TCPIP0::127.0.0.1::INSTR"

    ' If true, this will query the instrument model and fail initialization 
    ' if the model is not supported by the driver
    Const idQuery As Boolean = True

    ' If true, the instrument is reset at initialization
    Const reset As Boolean = True

    ' Setup IVI-defined initialization options
    Const standardInitOptions As String = "QueryInstrStatus=true, RangeCheck=true, Simulate=true"

    status = KtNA_InitWithOptions(resourceDescriptor, idQuery, reset, standardInitOptions, session)
    ' Driver task code here
    status = KtNA_close(session)

Accessing Attributes

IVI-C driver attributes are accessed through a set of IVI-defined accessor functions. Each attribute type has two accessor functions: Get for reading attribute values, and Set for writing attribute values.

Note: The driver VBA wrapper functions use standard VBA parameter types instead of the VISA types indicated in the attribute accessor function names.

The standard Get attribute accessors for reading attribute values are:

  • GetAttributeViInt32
  • GetAttributeViInt64
  • GetAttributeViReal64
  • GetAttributeViBoolean
  • GetAttributeViString

Correspondingly, the standard Set attribute accessors for writing attribute values are:

  • SetAttributeViInt32
  • SetAttributeViInt64
  • SetAttributeViReal64
  • SetAttributeViBoolean
  • SetAttributeViString

Each attribute accessor takes an attribute ID that uniquely identifies the attribute to access. These attribute IDs are global constants listed in the KtNA.bas wrapper file (and documented in Attributes by Name). The following example shows how to use attribute accessors to read and write driver attribute values.

Note: This code is for illustration only and may not apply to your driver.

VBA Copy Code
    Dim status As Long

    ' Set the value of an attribute
    status = KtNA_SetAttributeViReal64(session, "", KTNA_ATTR_VOLTAGE_RANGE, 20.0)

    ' Get the value of an attribute 
    Dim voltageRange as Double
    status = KtNA_GetAttributeViReal64(session, "", KTNA_ATTR_VOLTAGE_RANGE, voltageRange)

    ' Set the value of an attribute (enum).  Note use of the defined constant for the attribute value
    status = KtNA_SetAttributeViInt32(session, "", KTNA_ATTR_TEMP_TRANSDUCER_TYPE, KTNA_VAL_THERMOCOUPLE)

Reading String Values

String variables used with driver functions that return string values must be sized greater or equal to the expected string length, plus 1. These functions may include a buffer size parameter or use the string variable length to allocate memory for the returned string value. If the string parameter is not sized before it is used with the function, no memory is allocated for the returned value resulting in invalid results.

For example:

VBA Copy Code
    ' Strings used with driver functions must be sized >= expected string length + 1
    Const bufferSize As Integer = 1024
    Dim strVal   As String * bufferSize
    Dim errorMsg As String * bufferSize
    Dim errorCode As Long

    status = KtNA_GetAttributeViString(session, " ", KTNA_ATTR_INSTRUMENT_MODEL, bufferSize, strVal)
    MsgBox ("ATTR_INSTRUMENT_MODEL: " + strVal)

    status = KtNA_error_query(session, errorCode, errorMsg)
    MsgBox ("error_query: " + CStr(errorCode) + ", " + errorMsg)

Complete Example

Go to Windows Start Menu > All Programs > Keysight Instrument Drivers > IVI-C KtNA 3.0.8 to see more complete, working examples.

VBA Copy Code
Sub Kt_Example1()
    ' Requires Import File... <Program Files>\IVI Foundation\IVI\Include\KtNA.bas
    '
    ' Keysight IVI-C Driver VBA/Excel Example Program
    '
    ' Initializes the driver, reads a few Identity attributes, and checks the
    ' instrument error queue.  May include additional instrument specific functionality.
    '
    ' Runs in simulation mode without an instrument.
    '
    Dim session As Long
    Dim status As Long
    Dim errorCode As Long
    Dim simulate As String
    Dim resourceDesc As String
    
    ' Strings used with IVI-C driver functions must be sized >= expected string length +1
    Const bufferSize As Integer = 1024
    Dim errorMsg As String * bufferSize
    Dim strVal   As String * bufferSize

    ' Clear output cells
    Sheet1.Range("B9:B50").Clear
    Application.Wait (Now + TimeValue("0:00:01"))
    
    ' Get Inputs
    simulate = Sheet1.Cells(4, 2)
    resourceDesc = Sheet1.Cells(5, 2)
    ' Run in Simulation mode if either input is blank.
    If simulate = "" Or resourceDesc = "" Then simulate = "TRUE"

    ' Initialize driver
    Const idQuery As Boolean = True
    Const reset   As Boolean = True
    status = KtNA_InitWithOptions(resourceDesc, idQuery, reset, "QueryInstrStatus=True, Simulate=" + simulate, session)
    If status <> 0 then
        errorcode = status
        status = KtNA_error_message(session, errorcode, errormsg)
        MsgBox ("KtNA_InitWithOptions error " + CStr(errorCode) + " , " + errorMsg)
        End
    End If

    ' Output some identity attributes
    status = KtNA_GetAttributeViString(session, " ", KTNA_ATTR_SPECIFIC_DRIVER_PREFIX, bufferSize, strVal)
    Debug.Assert status = 0
    Sheet1.Cells(9, 2) = strVal
    status = KtNA_GetAttributeViString(session, " ", KTNA_ATTR_SPECIFIC_DRIVER_REVISION, bufferSize, strVal)
    Debug.Assert status = 0
    Sheet1.Cells(10, 2) = strVal
    status = KtNA_GetAttributeViString(session, " ", KTNA_ATTR_SPECIFIC_DRIVER_VENDOR, bufferSize, strVal)
    Debug.Assert status = 0
    Sheet1.Cells(11, 2) = strVal
    status = KtNA_GetAttributeViString(session, " ", KTNA_ATTR_SPECIFIC_DRIVER_DESCRIPTION, bufferSize, strVal)
    Debug.Assert status = 0
    Sheet1.Cells(12, 2) = strVal
    status = KtNA_GetAttributeViString(session, " ", KTNA_ATTR_INSTRUMENT_MODEL, bufferSize, strVal)
    Debug.Assert status = 0
    Sheet1.Cells(13, 2) = strVal
    status = KtNA_GetAttributeViString(session, " ", KTNA_ATTR_INSTRUMENT_FIRMWARE_REVISION, bufferSize, strVal)
    Debug.Assert status = 0
    Sheet1.Cells(14, 2) = strVal


    ' TODO: Typical instrument task
    status = KtNA_reset(session)
    Debug.Assert status = 0


    ' Check instrument error queue
    errorCode = 1
    errorMsg = " "
    Sheet1.Cells(18, 2) = " "
    While (errorCode)
        status = KtNA_error_query(session, errorCode, errorMsg)
        Debug.Assert status = 0
        Sheet1.Cells(18, 2) = Sheet1.Cells(18, 2) + CStr(errorCode) + ", " + errorMsg + " ; "
    Wend

    ' close and exit
    status=KtNA_close(session)
    Debug.assert status = 0
End Sub
