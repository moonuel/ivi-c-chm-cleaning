Driver Source Code Files

This section describes the files that are included with the source code and explains how they relate to each other.

There are multiple references to Nimbus. This driver was developed using Nimbus Driver Studio, and many of the driver source code files are either provided as part of Nimbus or generated (partially or wholly) by Nimbus. However, the source code was designed to be used without installing Nimbus Driver Studio. Everything that you need to use the source code is included in the installation.

Driver source code is not installed by default — it is selected and installed during driver installation. To install the source code later:

  1. Select Programs and Features from the Windows Control Panel.
  2. In the list of programs, select the KtNA driver.
  3. Click Change to modify the driver installation.

All IVI drivers are installed in the standard IVI Foundation drivers directory, typically:

  • C:\Program Files\IVI Foundation\IVI (64-bit systems)
  • C:\Program Files (x86)\IVI Foundation\IVI (32-bit systems)

Refer to the Installation topic for information about the directory. The driver source code is installed in the Source subdirectory of the Keysight.KtNA 3.0.8 driver directory.

The Source Directory

The Source directory includes:

  • All files needed to rebuild the driver dll
  • RCL (Reusable Class Library), which contains Keysight driver class library header and source files used by the driver to control the instrument via an external external C++ dll library (CppApi).

After building the driver, the Source directory also includes \bin and \obj, the standard directories Visual Studio uses during the build process.

The following table describes the files in the Source directory that you might reasonably expect to edit when modifying the driver. Other files in the source, RCL, and its subdirectories are needed to build the driver, but should not be modified when making changes related to instrument functionality.

File Name Description
KtNADriver.vcxproj Visual Studio 2010 project file for the driver (open in Visual Studio 2010 or higher).
KtNA.cpp Implementation code for the driver's functionality. To modifiy the driver, this is the file to edit.
RepCaps.h Implementation code for any parameter-style repeated capabilities.
Enums.h Enumerations defined in the KtNA driver. The file does not include enumerations defined in by IVI Inherent or IVI Instrument Classes.
DriverSession.h Functionality to manage the driver session class that is instantiated each time the driver is initialized.
Utility.h Helper methods and variables used internally by the driver.
Models.cpp Defines the instrument models and model families supported by this driver.
Errors.cpp Defines the instrument error codes and messages supported by this driver.
ConfigStore.xml Valid IVI Configuration Store that includes the entries to be made when this driver is installed (file is for reference only - do not modify).

The nrt.h File

The Source\nrt.h file contains the Nimbus Runtime Library (NRT). The NRT library is:

  • A class/template library that provides a variety of support functions for IVI-C compliant driver development
  • Part of the Pacific MindWorks Nimbus Driver Studio (version 4.0 or above) IVI driver development tool
  • Covered by the license terms contained in the Keysight EULA and the Keysight IVI Drivers Supplemental License Terms.rtf file

Each driver comes with its own copy of the NRT library source, so there is never a question as to which version of the library is being used. Consequently, the NRT code for the KtNA driver can be modified without affecting other installed drivers. However, Keysight recommends that you do NOT edit NRT code because subtle interconnections exist and it is easy to compromise IVI-C compliance when modifying NRT code.

The KtNA driver does not use all of the functionality in the NRT library. For example, the Cache attribute is typically not used in Keysight IVI-C drivers because of difficulty in implementing caching correctly. Also, many of the NRT attributes are designed for use in the NRT library code only, and should not be used by driver developers. In these cases, the attribute source code file usually has a comment to that effect.

The RCL_NET Directory

The Source\RCL directory and sub-directories contains the Keysight Reuseable .NET Class Library (RCL) files. The RCL library is:

  • A library of header and implementation files required to build the IVI-C driver source. These files allow the IVI-C driver to call the underlying CppApi dll that controls the instrument including marshalling of various data types to/from the instrument control dll.
  • Covered by the license terms contained in the Keysight EULA and the Keysight IVI Drivers Supplemental License Terms.rtf file

Each driver comes with its own copy of the RCL library source, so there is never a question as to which version of the library is being used. Keysight recommends that you do NOT edit RCL code because subtle interconnections exist between the IVI-C driver and its CppApi dll that could be compromized by editing resulting in corrupt data or unexpected behavior.
