Using Driver Source Code

This topic provides an introduction to using the KtNA driver source code. See the subtopics for more information:

  • Driver Source Code Files describes the source code files and explains how they relate to each other
  • Modifying the Driver explains how to build the source code and deploy the new driver assembly to other systems

Introduction to Source Code

Driver source code is not installed by default — it is selected and installed during driver installation. To install the source code later:

  1. Select Programs and Features from the Windows Control Panel.
  2. In the Programs and Features list, select the KtNA driver.
  3. Click Change to modify the driver installation.

Visual Studio 2017 or greater is required to build the driver. You may also need an active internet connection the first time you build the driver. This is needed for the build to automatically download the Nimbus Build Tools NuGet package, which is required for building. An internet connection will not be required for subsequent builds.

This driver uses an external C++ dll library (CppApi) which performs the instrument I/O and contains the SCPI commands and queries used to control the instrument.

Several techniques are available to see the SCPI mnemeonics sent by a driver item:

  • View the item's help page or refer to the Instrument Command Reference page in this help file.
  • Enable the driver's tracing functionality using the SYSTEM_TRACING_ENABLED attribute with SYSTEM_TRACING_LEVEL set to Debug.
  • Use the Keysight I/O Libraries Suite, I/O Monitor utility to monitor the VISA traffic.

Note that, in general:

  • Keysight provides source code for drivers that communicate with devices using message-based protocols. SCPI has a relatively expressive, friendly syntax and has become a staple of message-based communication over the last twenty-five years.
  • Keysight does not provide source for modular instruments (such as PXI or AXIe instruments). These instruments use register-based I/O to manipulate instrument registers. Register-based I/O is relatively low-level, non-intuitive, and not stable from revision to revision of an instrument, and so the source is not useful.

There are license restrictions on the use of the source code. For details, see the Keysight IVI Drivers Supplemental License Terms.rtf file installed in the main Keysight.KtNA 3.0.8 driver directory.
